Files and Folders Structure

In your zip archive you have 3 folders: "Admin", "Documentation" and "Frontend-Template"

Admin folder: here are all files required for administration area. Upload this folder to your server. Optionaly, you can rename this folder (do not forget to change folder name in config.php file if you rename it).

FrontEnd-Template folder. Here you have all website template files. You can upload files inside this folder to your location where you want to install website template (in root, in separate folder or subdomain). Please note that it is a separate config.php file that you must edit. It is not required to use our template but it is a good start if you want to build a complex template that will use content added in our Pike Admin administration area.

Requirements

- PHP 5.3+ with pdo extension enabled (it is default enabled)
- MySQL database
- Web server with ModRewrite support (if you use Apache server, you have a .htaccess file which you must upload)
- Script require a domain name (or subdomain) to be installed.

Installation

STEP 1: Unzip and copy script files and folders on the server (using an FTP program).

STEP 2: Create a database and a user for that database (eg. in cPanel -> MySQL Databases)

STEP 3: Import tables file (/admin/install/tables.sql). Steps to import a file: - go to phpMyAdmin (or another database management script) - select your database - click on "Import" button - choose file to upload (located in /admin/install/tables.sql) - upload (import) file

STEP 4: Edit the file "config.php" located in "admin" folder and add database connect details and url where you uploaded the "admin" folder.

STEP 5: Make writable (chmod 777) folder "uploads" folder and all subfolders ("avatars", "images", "large", "small", "temp")

STEP 6: Login in admin area. Default admin connection details are: demo@pikephp.com (as email) and 123456 (as password). You can change this details in admin area (my profile).

STEP 7 (required for frontend-template only): If you want to use our default front-end template, you can upload files inside "frontend-template" to your server. Don't forget to upload .htacess file (located in root of "frontend-template"folder). Also you need to edit settings in config.php file located in "frontend-templae" folder too.

IMPORTANT:
- It is strongly recomended to use a SMTP server to manage sending emails (related to users registration, forget password, activate account...). We recomend using Mailgun SMTP server.
- It is strongly recomended to use Google reCAPTCHA API to prevent spam related to users registration and contact messages. Get reCAPTCHA keys.
If you want support to setup and integrate this products in your software, you can contact us (note that this will be a paid support).

Files and Folders Structure for "admin" area

"assets" folder. Here are all assets files (css styles, javascripts, images, fonts, plugins...)

"core" folder. Here you have all core php files that will process content related to users, registration, articles, contact messages and so...

"template-files" folder. Here you have all template files in php format.

"upload" folder. Here you have all media content added in admin area (user avatars, article images, slider images...)

Files in root admin folder:
- index.php - will handle login area (login, registration, reset password...);
- account.php - will handle adminsitartion area after you login;
- global-head.php - will handle all html content in head area that will be loaded in all admin pages;
- global-footer.php - will handle all footer html content that will be loaded in all admin pages;
- navigation.php - will handle top horizontal navigation inside admin area;
- sidebar.php - will handle left vertical sidebar where you have links to all sections inside admin area;

FAQ

I want to edit / delete or add pages in admin area

Admin pages are located in "templates-pages" folder inside admin folder. Left sidebar (with all admin sections links) are located in "sidebar.php" file.

If you add / edit a page, keep in mind that the same filename must be present in sidebar.php

Example: in sidebar if you have a link like 

... href="account.php?page=my-custom-page ...
The filename inside "templates-pages" must be "my-custom-page.php"

Users roles

We didn't implement users roles and separate template files depending on permission. It is your responsability to code this. All users roles have access to all pages.
